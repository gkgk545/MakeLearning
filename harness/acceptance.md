# 하네스 합격 기준 (Acceptance Criteria)

스킬이 생성한 게임과 스킬의 행동이 모두 아래 기준을 충족해야 해당 샘플 실행을 '합격'으로 판정한다.

## 1. 산출물 기준 (게임 HTML)

`skill/make-learn/core/verify-rubric.md`의 전 항목(A1~A5, B1~B6, C1~C5, D1~D5 + 패턴별 검증 추가 포인트)을 **독립 검증자**(생성한 에이전트가 아닌 별도 실행)가 재점검해 통과해야 한다. 추가로:

| # | 기준 |
|---|------|
| P1 | `node --check`(또는 동등한 방법)로 JS 문법 오류 0건 |
| P2 | 파일 내 `http` 검색 시 fonts.googleapis.com / fonts.gstatic.com 외 외부 URL 0건 |
| P3 | 브라우저 실 구동: 콘솔 오류 0건, 시작→진행(정답·오답 각 1회 이상)→결과 화면 도달 |
| P4 | 390px(모바일)·1280px(데스크톱) 스크린샷에서 겹침·잘림·화면 밖 요소 없음 |
| P5 | 줄 수 600~1,200 범위 (±20% 허용, 크게 벗어나면 사유 확인) |

## 2. 행동 기준 (스킬 파이프라인)

| # | 기준 |
|---|------|
| S1 | 자료 분석 요약을 교사에게 확인받음 (학교급·교과·핵심 개념이 샘플의 의도와 일치) |
| S2 | 인터뷰 질문 수 ≤5 (최대 7), 모든 질문에 추천안 포함, "추천대로"만으로 완주 가능 |
| S3 | 게임 유형 추천이 유형 적합 요약표 기준에 부합 (예: 개별 접속용에 팀 점수판을 추천하면 불합격) |
| S4 | 게임 명세서를 제시하고 승인 후에만 제작 시작 |
| S5 | 문항이 전부 샘플 자료 범위 내 (규칙 기반 생성은 명세서에 명시된 경우만) |
| S6 | 7단계 안내문에 사용법·공유 방법(주 활용 방식에 맞는 것만)·문항 수정 방법 포함 |
| S7 | 교사 대상 출력에 기술 용어 없음 (localStorage, 반응형, 뷰포트 등) |

## 3. 샘플별 기대 속성

| 샘플 | 기대 분석 결과 | 적합 유형(이 중 하나면 통과) |
|------|---------------|------------------------------|
| elem-math (분수 덧뺄셈) | 초4 수학 | 04 스피드, 01 퀴즈 |
| elem-korean (맞춤법) | 초3 국어 | 01 퀴즈(OX/2지선다), 04 스피드, 06 낱말 |
| mid-science (광합성) | 중1 과학 | 05 팀 대항, 01 퀴즈, 02 짝맞추기, 06 빈칸 |
| mid-english (Lesson 4 어휘) | 중1 영어 | 02 짝맞추기, 06 낱말, 04 스피드 |
| high-social (기회비용) | 고1 통합사회 | 01 퀴즈, 06 빈칸, 02 짝맞추기 |
| elem-practical (식품군) | 초5 실과 | 03 분류, 02 짝맞추기, 01 퀴즈 |

디자인 톤: elem-* 은 초등 톤, mid-* 는 중등 톤, high-* 는 고등 톤이 적용되어야 한다 (D5).

## 4. 전체 합격 판정

- **릴리스 기준**: 6종 샘플 전부 1·2·3 기준 통과 + 3개 플랫폼 설치 테스트(test-protocol.md) 통과.
- 불합격 발견 시: 원인을 산출물(개별 버그)인지 스킬 문서(지침 결함)인지 구분해 기록하고, 스킬 문서 결함이면 core 문서를 수정한 뒤 해당 샘플을 재실행한다.
