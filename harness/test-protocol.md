# 플랫폼별 설치·실행 테스트 절차

스킬 수정 후 또는 릴리스 전에 수행하는 점검 절차.

## 0. 사전 준비

- 저장소 최신 상태에서 `bash install.sh`(macOS/Linux) 또는 `install.ps1`(Windows) 실행.
- 설치 확인:
  - `~/.claude/skills/make-learn/SKILL.md` 존재
  - `~/.codex/skills/make-learn/SKILL.md` 존재
  - `~/.make-learn/SKILL.md` 존재
  - `~/.gemini/antigravity/global_workflows/make-learn.md` 존재

## 1. 공통 테스트 시나리오

각 플랫폼에서 동일하게 수행:

1. 새 세션 시작 (재시작 후), 빈 작업 폴더에서.
2. `/make-learn` 입력 + `harness/samples/` 중 샘플 1개 첨부(또는 경로 제시).
3. 교사 역할 응답: 첫 질문에 활용 방식 하나를 답하고, 이후 "추천대로 해줘" → 명세서에 "좋아요".
4. 완성까지 대기 후 기록:

| 점검 | 통과 기준 |
|------|----------|
| T1 스킬 인식 | /make-learn 입력 시 스킬/워크플로우가 호출됨 |
| T2 코어 문서 접근 | core 문서를 찾지 못하는 오류 없음 |
| T3 파이프라인 준수 | 분석 요약 → 인터뷰(≤5문항, 추천 포함) → 명세서 승인 → 제작 → 검증 보고 → 안내문 순서 |
| T4 산출물 | HTML 파일 생성, 더블클릭으로 열어 1분 플레이 정상 |
| T5 합격 기준 | acceptance.md 1·2절 점검 |

## 2. 플랫폼별 특이사항 점검

### Claude Code
- 스킬 자동 트리거도 확인: "/make-learn" 없이 "이 자료로 복습 게임 만들어줘"라고 해도 스킬이 발동하는지 (description 매칭).
- 브라우저 테스트 도구(webapp-testing)가 있으므로 6단계에서 실 구동 검증을 수행했는지 확인.

### Codex
- 호출: 스킬 목록에서 make-learn 확인 (`/make-learn` 또는 스킬 자동 매칭).
- `/goal` 기능이 켜진 환경이면: 4단계 승인 후 goal 등록을 시도하거나 안내하는지 관찰 (필수는 아님 — 직접 제작 폴백도 통과).
- PDF 샘플 입력 케이스 1회 포함 권장 (플랫폼별 파일 읽기 차이 확인).

### Antigravity
- Agent Manager에서 `/make-learn` 입력 시 전역 워크플로우가 뜨는지 확인.
- 워크플로우가 `~/.make-learn/SKILL.md`를 정상적으로 읽는지 (T2).
- 미설치 상태 메시지 테스트: `~/.make-learn`을 임시로 이름 변경 후 호출 → "설치 스크립트를 먼저 실행해 주세요" 안내가 나오는지. (테스트 후 원복)

## 3. 결과 기록 양식

```
일시 / 플랫폼·버전 / 샘플 / 활용 방식 답변
T1~T5: 통과 여부 (실패 시 증상)
스킬 문서 결함 발견: (있으면 구체적으로 → core 문서 수정 후 재실행)
산출물 평가: acceptance.md 기준 항목별
```

기록은 `harness/runs/` 폴더에 `YYYYMMDD-플랫폼-샘플.md`로 저장한다.

## 4. 회귀 원칙

core 문서를 수정하면 최소한 (a) 수정과 관련된 샘플 1종 (b) 활용 방식이 다른 샘플 1종을 재실행해 회귀를 확인한다.
