# 패턴 01 — 골든벨 퀴즈 (선택형)

문제가 한 번에 하나씩 나오고 4지선다 또는 OX로 답하는 가장 범용적인 패턴.

## 이런 자료에 적합
- 사실·개념 확인 문항을 만들기 좋은 거의 모든 자료 (가장 안전한 기본 선택지)
- 정의·특징·인과관계가 명확한 내용 (예: 과학 개념, 사회 제도, 국어 문법, 도덕 가치 판단 OX)
- 자료에서 8문항 이상 뽑을 수 있을 때

## 게임 규칙
1. 문제가 하나씩 제시되고, 보기 4개(또는 O/X) 중 정답을 고른다.
2. 정답이면 점수 획득(+연속 정답 보너스 선택), 오답이면 정답·해설 표시 후 다음 문제.
3. 모든 문제 후 결과 화면: 점수, 등급(예: 🥇 골든벨!/🥈 도전자/💪 다시 도전), 틀린 문항 다시 보기.

## 화면 흐름
시작(제목·규칙·시작 버튼) → 문제 루프(진행도 ○/○, 문제, 보기, [선택 → 피드백 → 다음]) → 결과

## GAME_DATA 스키마
```javascript
const GAME_DATA = {
  title: "게임 제목",
  grade: "초등 5학년 과학",
  type: "choice",            // "choice"(4지선다) | "ox"
  questions: [
    { q: "문제", choices: ["보기1","보기2","보기3","보기4"], answer: 0, explain: "해설" },
    // OX형: { q: "문제", answer: true, explain: "해설" }
  ]
};
```

## 구현 포인트
- 문항 순서는 매 판 셔플. 4지선다는 보기도 셔플하되 answer 매핑 유지 (보기 객체를 통째로 섞고 정답 텍스트로 비교하는 방식이 안전).
- 보기 선택 후 즉시 채점: 선택한 보기 색 변경(정답 초록/오답 빨강) + 정답 보기 강조 + 해설 박스 → "다음" 버튼으로 진행 (자동 진행보다 교사·학생이 해설 읽을 시간 확보).
- 채점 후에는 보기 클릭 비활성화 (중복 채점 방지).

## 활용 방식별 변형
- **TV 수업용**: 보기 선택 대신 **'정답 공개' 버튼** 방식 — 문제를 띄우고 학생들이 구두로 답한 뒤 교사가 클릭하면 정답·해설 공개. 점수는 선택 사항(학급 전체 도전 횟수 표시 등). 글자 크게. **'틀린 문항 다시 보기'(D3)를 유지하려면 정답 공개 후 교사가 [맞혔어요/못 맞혔어요]를 판정하는 버튼을 두는 방식을 권장** — 못 맞힌 문항이 결과 화면 복습 목록에 쌓인다.
- **개별 접속용**: 위 기본 규칙 그대로. 모바일 세로 레이아웃, 보기 버튼을 세로로 큼직하게.
- **전자칠판용**: 기본 규칙 + 보기 버튼을 화면 하단에 대형으로 배치.

## 검증 추가 포인트
- 보기 셔플 후에도 정답 판정이 올바른가 (A2 연계)
- 오답 보기들도 자료 범위 내에서 그럴듯하게 구성됐는가 (전혀 무관한 보기로 난이도가 무의미해지지 않게)
