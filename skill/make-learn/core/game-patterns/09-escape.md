# 패턴 09 — 방탈출 미션

스토리로 몰입시킨 뒤 미션을 순서대로 잠금 해제하며 풀고, 미션마다 얻은 자물쇠 숫자 조각을 모아 최종 자물쇠를 여는 방탈출 게임.

## 이런 자료에 적합
- **단원 전체를 아우르는 자료** — 여러 소주제가 한 단원에 묶여 있고, 그것을 종합 복습하고 싶을 때
- 단원 마무리·중간고사 직전 종합 점검처럼, 한 게임 안에서 여러 영역을 차례로 짚고 싶을 때
- 문항을 9~15개 뽑을 수 있을 때 (미션 3~5개 × 미션당 문항 2~3개)
- 한 판에 **완결되는 서사·달성감**이 동기 부여에 효과적인 학급
- 단일 주제·소수 문항(8문항 이하)이라면 패턴 01(골든벨)이 더 가볍다 — 이 패턴은 분량이 있어야 산다.

## 게임 규칙
1. 시작 시 **스토리 인트로**(한두 줄)로 상황을 깔아 몰입시킨다 — "갇힌 방에서 탈출하려면 자물쇠 비밀번호를 알아내야 한다".
2. 미션 3~5개가 세로로 나열되되, **첫 미션만 열려 있고 나머지는 잠금 상태**다. 미션은 위에서부터 순서대로만 도전할 수 있다.
3. 한 미션에 문항이 1개 이상 들어 있다. **미션 안의 모든 문항을 통과해야** 그 미션이 클리어된다. 오답이면 정답·해설을 보여준 뒤 그 문항을 다시 풀게 한다.
4. 미션을 클리어하면 **자물쇠 숫자 조각(clue) 한 자리를 획득**하는 연출이 뜨고, 다음 미션의 잠금이 풀린다.
5. 마지막 미션까지 클리어하면 **최종 자물쇠 키패드** 화면으로 전환된다. 모은 조각을 **미션 순서대로** 입력한다.
6. 입력값이 정답 코드와 일치하면 **탈출 성공** 결과 화면, 틀리면 "다시 확인" 안내 후 재입력(획득한 조각은 힌트로 다시 볼 수 있다).
7. 결과 화면: 탈출 성공 연출, 클리어 시간(선택), **틀렸던 문항 다시 보기**, 다시 하기.

## 구현 포인트
- **미션 순차 잠금**: 미션 목록은 `classList`로 잠금(`locked`)/해제(`open`)/완료(`cleared`) 상태를 토글한다. 잠긴 미션은 흐리게 + 자물쇠 아이콘, 클릭해도 진입 불가. 현재 도전 가능한 미션 하나만 활성화하면 학생이 헤매지 않는다.
- **미션 내 진행**: 한 미션을 열면 그 미션의 문항을 차례로 푼다. 문항은 **미션 단위로 셔플**한다(미션 간 순서는 잠금 해제 규칙상 고정 — 게임 흐름이 곧 잠금 순서다). 미션 내 각 문항의 **보기(choices)도 셔플**하되, 정답은 인덱스가 아니라 정답 텍스트로 판정해 매핑을 유지한다(셔플 전 `choices[answer]`로 정답 문자열을 잡아 두고 비교 — 01-quiz·10-deduction과 동일). 미션의 모든 문항을 통과해야 클리어로 처리한다.
- **clue 획득 연출**: 미션 클리어 시 그 미션의 `clue` 숫자가 자물쇠 조각으로 날아가 상단 **조각 모음(○○○○ 형태의 자리표시)**에 채워지는 연출. 획득한 조각은 항상 화면에 보이게 둔다(최종 입력 단서).
- **최종 자물쇠 키패드**: 마지막 미션 클리어 후 키패드 UI(0~9 + 지우기 + 확인)로 전환. 키패드 화면 상단에 `finalLock.prompt`를 안내 문구로 표시한다. 입력 자릿수는 `finalLock.code` 길이만큼 표시. 입력값이 `finalLock.code`와 **문자열 일치**하면 탈출 성공, 아니면 흔들림 효과 + "다시 확인하세요" 안내 후 재입력(시도 횟수 제한 없음, 조각 힌트 상시 노출).
- **데이터 정합 규칙**: `finalLock.code`는 반드시 `missions`의 `clue`를 **미션 순서대로 연결한 값**과 같아야 한다(예시: "7"+"3"+"1"+"9" = "7319"). 코드 안에서 `missions.map(m=>m.clue).join("")`로 자물쇠 정답을 산출하면 이 둘이 어긋날 일이 없다 — 권장 방식.
- **오답 처리**: 미션 문항 오답 시 design-guide 절대 규칙대로 정답+한 줄 해설(`explain`)을 보여준 뒤 같은 문항을 다시 풀게 한다(미션 통과 전까지는 다음 문항으로 넘어가지 않음).
- 효과음은 design-guide의 표준 `SoundEffect` 유틸을 쓴다 — 미션 클리어/조각 획득에는 `correct`, 자물쇠 해제 성공에 `correct`, 오답·자물쇠 불일치에 `wrong`을 매핑한다.

## 활용 방식별 변형
- **TV 수업용**: **학급 전체 협동 추리**. 교사가 진행하며 문항을 띄우고 학생들이 다 같이 답을 외치면 교사가 판정·진행. 모은 조각을 칠판처럼 크게 띄워 두고, 마지막에 학생들이 비밀번호를 함께 맞혀 키패드에 입력한다. 조각 모음과 키패드를 한 화면에 크게 수납.
- **개별 접속용**: **혼자 탈출 도전**. 학생이 각자 미션을 순서대로 풀고 키패드까지 완주. 클리어 시간 기록(localStorage)으로 재도전 동기를 준다. 모바일 세로에서 미션 목록·키패드를 세로로 큼직하게.
- **전자칠판용**: **모둠 릴레이로 미션 분담**. 미션 1은 1모둠, 미션 2는 2모둠 식으로 나와서 직접 터치해 풀고, 마지막 자물쇠는 전체가 함께 입력. 키패드 버튼을 대형 터치 타겟으로.

## 검증 추가 포인트
- `finalLock.code`가 미션들의 `clue`를 **미션 순서대로 이은 값과 정확히 일치**하는가 (불일치면 탈출 자체가 불가능 — B6 연계)
- 미션 잠금/해제 순서가 정확한가 — 이전 미션 미클리어 시 다음 미션 진입이 막히고, 클리어 시에만 다음이 열리는가 (전체 흐름이 잠금 순서대로 진행되는지 B1 연계 + 잠금 상태 클래스 토글 정합성 B7 연계)
- 미션의 **모든 문항을 통과해야** 클리어로 처리되는가, 한 문항만 맞혀도 clue가 새지 않는가
- 최종 자물쇠 **오답 입력 시 재시도** 처리가 깔끔한가 — 입력값 초기화, 흔들림 후 정상 재입력, 정답 시에만 성공 전환 (B4 연계)
- 모든 미션의 모든 문항이 학습자료 범위 안에서 출제됐는가 (A1 연계)
- **마지막 미션 클리어 후** 자물쇠 키패드 화면으로 빠짐없이 전환되는가 (off-by-one 없이 — B2 연계)
