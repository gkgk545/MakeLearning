# 패턴 04 — 스피드 챌린지

제한 시간 안에 최대한 많은 문제를 푸는 게임. 속도와 콤보가 동기를 만든다.

## 이런 자료에 적합
- **즉답 가능한 짧은 문항**이 많이 나오는 자료 — 연산(수학), 단어 뜻(영어), 맞춤법 OX(국어), 용어 확인, 음계 읽기(음악)
- 반복 훈련으로 자동화가 목표인 내용 (이해보다 숙달)
- 문항을 15개 이상 뽑을 수 있을 때 (반복 출제 허용 시 그 이하도 가능)

## 게임 규칙
1. 제한 시간(기본 60~90초) 동안 문제가 연속 제시된다.
2. 정답: +10점, 콤보 보너스는 3연속 정답부터 문항당 **고정 +5점** (누진 아님). 오답: 콤보 초기화 + **정답·해설이 잠깐 표시**(시간은 계속 흐름 — 긴장감 유지하되 학습 노출 보장).
3. 시간 종료 시 결과: 점수, 푼 문제 수, 정답률, 최고 콤보, 틀린 문항 다시 보기, 최고 기록(localStorage).

## GAME_DATA 스키마
```javascript
const GAME_DATA = {
  title: "게임 제목",
  grade: "초등 4학년 수학",
  timeLimit: 60,             // 초
  comboBonus: true,
  questions: [
    { q: "7 × 8 = ?", choices: ["54","56","63","48"], answer: 1, explain: "7×8=56" },
  ],
  // 연산 게임은 generator 사용 가능: 문항을 코드로 생성 (범위를 자료 기준으로)
};
```

## 구현 포인트
- 문제 풀(pool)을 셔플해 순환 출제. 풀이 소진되면 다시 셔플 (단, 직전 문제 연속 반복 방지).
- 타이머는 항상 보이게 상단 고정, 마지막 10초는 색 변화·맥박 효과.
- 오답 해설 표시는 1.5~2초 오버레이로 — 진행을 멈추지 않되 반드시 보이게.
- 연산류는 GAME_DATA에 생성 규칙(범위·연산자)을 두고 코드로 무한 생성 가능. 단 범위는 학습자료 수준을 벗어나지 않아야 하고, 생성 사용 여부를 게임 명세서에 명시해 교사 승인을 받아야 한다 (core 5단계 예외 조항).
- 효과음(정답 '띵', 콤보 상승음)은 Web Audio 비프음으로, 켬/끔 버튼 제공.

## 활용 방식별 변형
- **TV 수업용**: 전체가 함께 도전하는 '우리 반 신기록' 모드 — 학생들이 외치면 교사가 클릭, 학급 최고 점수를 localStorage에 저장해 차시마다 갱신 도전.
- **개별 접속용**: 기본 규칙 그대로. 개인 최고 기록 표시가 동기 부여 핵심.
- **전자칠판용**: 릴레이 모드 운영 팁 — 한 명이 10초씩 풀고 다음 학생과 교대 (게임은 동일, 운영만 다름).

## 검증 추가 포인트
- 시간 종료 순간 입력 처리 (종료 후 클릭이 채점되지 않음 — B5 연계)
- 콤보·점수 계산 정확성 (오답 시 콤보 0 초기화)
- 문항 생성기 사용 시: 생성 범위가 자료 수준과 일치하는가 (A1 연계)
