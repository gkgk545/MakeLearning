# 패턴 03 — 분류·정렬 게임

항목을 올바른 범주 바구니에 넣거나(분류), 올바른 순서로 배열하는(정렬) 게임.

## 이런 자료에 적합
- **분류형**: 범주가 2~5개로 명확한 자료 — 식물/동물, 고체/액체/기체, 사실/의견, 현재형/과거형(영어), 식품군, 타악기/현악기/관악기, 권리/의무
- **정렬형**: 순서·과정이 핵심인 자료 — 식물의 한살이, 역사 연표, 요리 순서(실과), 실험 절차, 이야기 사건 순서(국어)
- 분류 항목 10~25개 또는 순서 단계 4~8개를 뽑을 수 있을 때

## 게임 규칙
**분류형**: 화면 위에서 항목 카드가 하나씩 나오고(또는 목록으로 있고), 아래 범주 바구니 중 맞는 곳에 넣는다. 맞으면 바구니에 쌓이고, 틀리면 정답 범주와 이유 표시.
**정렬형**: 뒤섞인 단계 카드를 올바른 순서로 배열한다. 완성 후 '확인'을 누르면 전체 판정, 틀린 위치는 표시 후 재시도(2회) 또는 정답 공개.

## GAME_DATA 스키마
```javascript
const GAME_DATA = {
  title: "게임 제목",
  grade: "초등 3학년 과학",
  variant: "classify",       // "classify"(분류) | "order"(정렬)
  // 분류형:
  categories: ["고체", "액체", "기체"],
  items: [
    { text: "얼음", category: 0, explain: "모양이 일정하니 고체예요" },
  ],
  // 정렬형: steps 배열을 올바른 순서로 기재 (게임이 알아서 섞음)
  steps: [
    { text: "씨앗을 심는다", explain: "" },
  ]
};
```

## 구현 포인트
- **조작은 탭-탭 방식을 기본**으로 한다 (드래그보다 전자칠판·저학년에서 훨씬 안정적). 제시 방식에 따라:
  - **한 항목씩 제시 모드(기본)**: 바구니 탭 한 번으로 판정해도 되고, '카드 탭(집기) → 바구니 탭' 2단계로 해도 된다. 2단계는 전자칠판에서 참여감을 높이는 연출용 — 어느 쪽인지 명세서에 정한다. 1단계 방식이면 검증 포인트의 '선택 해제' 항목은 N/A.
  - **전체 목록 펼침 모드(고학년용)**: 항목 탭(선택/재탭 해제) → 바구니 탭 판정의 2단계가 필수.
- 분류형은 한 번에 한 항목씩 제시하는 방식이 집중도·피드백 면에서 유리 (전체 목록 펼침은 고학년용).
- 데스크톱·드래그가 자연스러운 경우 pointer 이벤트 드래그를 추가로 지원해도 좋다(필수 아님).
- 오답 시: 항목이 올바른 바구니로 이동하는 모션 + 이유 한 줄 → 학습 효과의 핵심.
- 정렬형 판정: 위치 단위 채점 (몇 개 맞았는지), 전체 정답 시 과정 전체를 순서대로 다시 보여주는 복습 화면.

## 활용 방식별 변형
- **TV 수업용**: 항목이 나오면 학생들이 범주를 외치고 교사가 클릭. 또는 모둠별로 돌아가며 답하게 하고 점수 기록(팀 점수판 결합 가능).
- **개별 접속용**: 기본 규칙. 모바일에서 바구니를 화면 하단에 고정 배치.
- **전자칠판용**: 탭-탭 방식 그대로 최적. 항목·바구니 모두 대형으로.

## 검증 추가 포인트
- 범주 경계가 모호한 항목이 없는가 (자료 기준으로 정답이 유일해야 함 — A5 연계)
- 정렬형: 동순위가 가능한 단계가 없는가 (예: 동시에 일어나는 두 사건)
- 탭-탭 상태 관리: 선택 해제(같은 항목 재탭), 연속 탭 꼬임 없음 (B4 연계)
