# 패턴 10 — 힌트 추리 (다섯고개)

정답 대상을 숨긴 채 힌트를 추상→구체 순으로 하나씩 공개하고, 적은 힌트로 맞힐수록 높은 점수를 주는 추리 게임.

## 이런 자료에 적합
- 한 대상의 **특징을 여러 개 나열할 수 있는** 자료 6~12개 — 역사 인물, 과학 개념·생물, 나라·도시, 문학 작품, 직업
- "이것은 무엇일까/누구일까"로 좁혀 가는 스무고개식 사고가 학습 목표인 내용
- 단순 사실 확인(01 골든벨)보다 **추론·연결 과정**을 강조하고 싶을 때
- 대상마다 추상적 단서에서 결정적 단서까지 3~5단계로 정리할 수 있을 때

## 변형 선택
- **choice(보기 선택)**: 라운드마다 보기 4개(정답 포함)를 주고 자동 채점 — 개별 접속·자동 채점에 적합.
- **teacher(교사 판정)**: 보기 없이 학생들이 구두로 답하고 교사가 [맞음/틀림]을 누르는 방식 — TV 수업·모둠 토론에 적합.

## 게임 규칙
1. 라운드마다 정답 대상이 하나 정해지고, 첫 힌트(가장 추상적)만 공개된 채 시작한다.
2. 학생은 지금까지 공개된 힌트만으로 정답을 추리한다.
   - **choice 모드**: 보기 중 하나를 고른다.
   - **teacher 모드**: 학생이 구두로 답하고 교사가 [맞음/틀림]을 누른다.
3. 맞히면 그 시점의 점수를 획득한다 — **첫 힌트에 맞히면 만점, 힌트를 더 볼수록 점수가 낮아진다.** 남은 힌트는 건너뛰고 정답·해설을 보여준 뒤 다음 라운드로 넘어간다.
4. 틀리면 **다음 힌트(더 구체적)를 공개**하고 점수를 한 단계 낮춘다. 다시 추리한다.
5. 마지막 힌트까지 공개했는데도 못 맞히면 **정답+해설을 공개**하고(완전 실패 → 0점), 못 맞힌 대상을 복습 목록에 넣는다.
6. 모든 라운드 후 결과 화면: 총점, 등급, **틀린(못 맞힌) 대상 다시 보기**(정답+힌트 모음+해설).

## GAME_DATA 스키마
```javascript
const GAME_DATA = {
  title: "이 인물은 누구일까",        // 게임 제목
  grade: "초등 5학년 사회",           // 대상
  judgeMode: "choice",                // "choice"(보기 선택 자동 채점) | "teacher"(교사 판정 — 구두 응답)
  rounds: [
    {
      answer: "세종대왕",             // 정답 대상
      hints: [                        // 힌트는 반드시 추상→구체 순 (3~5개)
        "조선 시대의 임금이에요.",                  // 가장 추상적 (첫 공개)
        "백성을 무척 아낀 왕으로 유명해요.",
        "여러 학자들과 함께 책을 만들었어요.",
        "우리 글자를 직접 만들어 반포했어요."        // 가장 결정적 (마지막)
      ],
      // choices는 judgeMode가 "choice"일 때만 — 반드시 answer를 포함한 4개
      choices: ["세종대왕", "이순신", "정조", "광개토대왕"],
      explain: "세종대왕은 1443년 훈민정음을 창제해 백성이 쉽게 글을 익히게 했어요."  // 해설
    }
    // judgeMode가 "teacher"면 choices를 넣지 않는다 (미사용 필드 금지 — B6)
  ]
};
```

## 구현 포인트
- **힌트 1개씩 공개**: 시작 시 첫 힌트만 보이고, 오답 시 또는 [다음 힌트 보기] 버튼으로 다음 힌트가 한 줄씩 누적 공개된다(이전 힌트도 화면에 남겨 단서를 종합하게). 공개 여부는 classList 토글로 처리.
- **점수는 남은 힌트 수에 비례**: 예) 힌트 4개면 1번째 정답 100점, 2번째 70점, 3번째 40점, 4번째 20점(마지막 단계 점수=최소 득점), 전부 본 뒤에도 못 맞히면 0점. 힌트 개수가 라운드마다 달라도 "남은 힌트 비율"로 일관 계산한다(첫 힌트=만점, 마지막 힌트에 맞히면 최소 득점, 그래도 못 맞히면 0점).
- **choice 모드**: choices는 **매 라운드 셔플**하되 answer 텍스트로 정답을 판정해 매핑을 유지한다(인덱스 고정 금지). 오답 보기 클릭 시 다음 힌트 자동 공개. 정답을 맞히면 남은 힌트는 건너뛴다.
- **teacher 모드**: 힌트를 한 단계씩 넘기며 학생이 답하게 하고, 교사가 [맞음]을 누르면 그 시점 점수 획득, [틀림]이면 다음 힌트 공개. 마지막 힌트 후 정답·해설을 공개하면 교사가 [맞음/틀림] 최종 판정 → 못 맞힌 라운드를 복습 목록에 누적.
- **조기 정답 처리**: 정답을 맞힌 즉시 남은 힌트 단계를 건너뛰고, 그 라운드의 점수·진행도(○/○)를 정확히 반영한 뒤 정답·해설을 보여주고 다음 라운드로 넘어간다.
- 라운드 순서는 매 판 셔플한다. 단, **각 라운드 내 힌트 순서는 추상→구체 의미 순서이므로 섞지 않는다**(고정 배치 예외).
- 효과음은 design-guide의 표준 SoundEffect 유틸을 그대로 쓴다 — 정답/오답(다음 힌트)/클릭만으로 충분.

## 활용 방식별 변형
- **TV 수업용**: 이 패턴이 **가장 빛나는 환경**. 힌트를 한 줄씩 띄우고 학급 전체가 토론하며 추리한 뒤 교사가 판정 → judgeMode "teacher" 권장. 누적 힌트가 한 화면에 스크롤 없이 들어오도록 큰 글자로 배치. 힌트가 라운드 진행에 따라 누적되는 구조라 화면 수납에 가장 취약하므로, 힌트 5개 누적 + 보기 4개가 1280×800과 1920×1080 모두에서 스크롤 없이 보이도록 `@media (max-height:1100px)` 이상으로 힌트 줄간격·패딩을 압축한다.
- **개별 접속용**: 혼자 힌트를 넘기며 추리하고 보기로 자동 채점 → judgeMode "choice". 모바일 세로에서 누적 힌트는 위, 보기 버튼은 하단에 큼직하게.
- **전자칠판용**: 대형 화면에 힌트를 띄우고 모둠 대표가 나와 보기를 직접 터치하거나 구두로 답함. 보기/판정 버튼을 화면 하단에 대형으로.

## 검증 추가 포인트
- 힌트 배열이 **추상→구체 순**으로 정렬돼 있는가 (첫 힌트만으로 바로 정답이 특정되지 않아야 추리 게임이 성립 — A5 연계)
- choice 모드에서 choices에 answer가 반드시 포함되고, 셔플 후에도 정답 판정이 올바른가 (A2 연계)
- 조기 정답 시 남은 힌트를 건너뛰고 입력이 잠기는가 (연타·중복 입력 차단 — B4 연계)
- 조기 정답 시 점수·진행도(○/○)와 다음 라운드·결과 전환이 정확히 반영되는가 (B2 연계)
- 힌트 최대 개수(5)를 모두 공개한 상태 + 보기/판정 버튼이 1280×800과 1920×1080 두 해상도에서 폴드 안에 보이는가 (D2 연계)
- teacher 모드에서 못 맞힌 라운드가 복습 목록에 정확히 누적되는가 (D3 연계)
- 마지막 라운드까지 마친 뒤 결과 화면으로 정상 전환되는가 (B2 연계)
- judgeMode에 맞는 필드만 들어 있는가 (choice면 teacher 전용 필드 없음, teacher면 choices 없음 — B6 연계)
