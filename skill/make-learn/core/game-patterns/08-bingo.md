# 패턴 08 — 학습 빙고

N×N 빙고판을 정답 후보로 자동 생성하고, 문제마다 해당하는 칸을 찾아 탭해 줄을 완성하는 게임.

## 이런 자료에 적합
- 용어·짧은 정답이 9~25개 이상 나오는 자료 — 어휘, 인물, 용어 정의, 사회·과학 핵심어
- 정답이 한 낱말/짧은 구로 떨어져 칸 라벨로 쓸 수 있을 때 (긴 문장 정답은 패턴 01 권장)
- 같은 자료로 학생마다 다른 판을 주고 싶을 때 (개별 접속용에서 특히 강점)
- `cells` 후보를 `bingoSize`의 제곱 개수 이상 뽑을 수 있을 때 (3×3이면 9개+, 4×4면 16개+, 5×5면 25개+)

## 변형 선택
- 판 크기 `bingoSize`(3·4·5)는 자료의 정답 후보 개수와 학교급으로 정한다 — 저학년·짧은 수업은 3×3, 일반 복습은 4×4, 후보가 풍부하고 시간이 넉넉하면 5×5.
- 메커니즘 자체는 한 가지(문제→정답 칸 탭→줄 완성)로 동일하다. 패턴 01(정답 선택)과 달리 보기가 아니라 **판 위의 칸**이 정답 후보 풀이라는 점이 핵심 차이다.

## 게임 규칙
1. 게임 시작 시 `cells` 후보를 셔플해 `bingoSize`²개를 판에 배치한다(학생/판마다 배치가 다름, 남는 후보는 미사용).
2. 문제가 하나씩 제시되면 학생이 정답에 해당하는 칸을 찾아 탭한다.
3. 맞으면 그 칸이 마킹(도장)되고, 틀리면 정답 칸을 강조해 안내하고 한 줄 해설을 표시한다 — **틀린 경우 칸은 마킹하지 않는다.**
4. 마킹된 칸으로 가로·세로·대각선 줄을 완성한다. 완성 줄 수가 `targetLines`에 도달하면 즉시 '빙고!' 승리.
5. 모든 문제를 풀어도 빙고에 도달하지 못하면 마킹 수·완성 줄 수로 결과를 낸다. 결과 화면에서 놓친/틀린 문항을 **문제+정답+해설**로 다시 보기.

## GAME_DATA 스키마
```javascript
const GAME_DATA = {
  title: "조선의 인물 빙고",        // 게임 제목
  grade: "초등 5학년 사회",         // 대상
  bingoSize: 4,                     // 판 크기: 3(3×3) | 4(4×4) | 5(5×5)
  targetLines: 1,                   // 이 줄 수를 완성하면 '빙고!' 승리
  // 칸에 들어갈 정답 후보 — bingoSize의 제곱 개수 이상 (4×4면 16개 이상)
  // 셔플해서 16칸에 배치하고, 남는 후보는 그 판에서 미사용
  cells: [
    "세종대왕", "이순신", "신사임당", "정약용",
    "장영실", "허준", "김홍도", "유관순",
    "안중근", "이황", "이이", "정조",
    "황진이", "김정호", "최무선", "문익점",
    "박지원", "정도전"
  ],
  // 각 문제의 answer는 반드시 cells 중 하나와 정확히 일치해야 함
  // 한 줄(bingoSize칸) 완성에 필요한 칸 수 이상을 맞힐 수 있도록 문항을 충분히 둔다
  questions: [
    { q: "한글(훈민정음)을 창제한 조선의 임금은?", answer: "세종대왕", explain: "세종대왕은 1443년 훈민정음을 만들었어요." },
    { q: "거북선으로 임진왜란 해전을 이끈 장군은?", answer: "이순신", explain: "이순신은 한산도 대첩 등에서 크게 승리했어요." },
    { q: "「동의보감」을 지은 조선의 의관은?", answer: "허준", explain: "허준의 「동의보감」은 동아시아 의학의 보배예요." },
    { q: "대동여지도를 만든 지리학자는?", answer: "김정호", explain: "김정호는 정확한 우리나라 지도 대동여지도를 완성했어요." },
    { q: "자격루(물시계)를 만든 조선의 과학자는?", answer: "장영실", explain: "장영실은 자격루·앙부일구 등을 만들었어요." },
    { q: "「목민심서」를 쓴 조선 후기 실학자는?", answer: "정약용", explain: "정약용은 수원 화성 설계에도 참여한 실학자예요." }
    // 문항 수는 자유. 단, 한 줄(bingoSize칸)을 채울 수 있을 만큼 답이 cells에 골고루 들어가
    // 실제로 '빙고!'가 가능하도록 구성한다. targetLines를 높이면 그만큼 문항도 늘려야 함
  ]
};
```

## 구현 포인트
- **판 생성**: `cells`를 셔플해 앞에서 `bingoSize`²개만 잘라 격자에 배치(CSS Grid, `repeat(bingoSize, 1fr)`). 칸 라벨은 후보 문자열. 남는 후보는 그 판에서 쓰지 않는다. **문항 출제 순서도 매 판 셔플한다(design-guide 8절).**
- **정/오답 판정**: 현재 문제의 `answer`와 **라벨이 일치하는 칸을 탭했는지**로 판정한다(칸 인덱스가 아니라 라벨 비교 — 배치가 매 판 달라지기 때문). 이미 마킹된 칸을 다시 탭하면 무시.
- **줄 완성 검사**: 매 마킹 후 행(`bingoSize`개)·열(`bingoSize`개)·양 대각선(2개)을 모두 검사해 전부 마킹된 줄 수를 센다. 그 수가 `targetLines` 이상이면 승리 처리.
- **종료 처리 두 갈래**: ① 줄 수가 `targetLines` 도달 → 즉시 '빙고!' 결과(남은 문제는 건너뜀). ② 문제 소진 → 완성 줄 수·마킹 수로 결과. 둘 중 먼저 오는 쪽으로 끝난다.
- **오답 연출**: 정답 라벨 칸을 잠깐 강조(테두리·흔들림)하고 해설 박스 표시 → '다음' 버튼. 강조만 하고 마킹은 하지 않는다(틀렸으므로).
- 진행 중 화면에 현재 완성 줄 수 / `targetLines`를 표시해 목표를 보이게 한다.

## 활용 방식별 변형
- **개별 접속용**: 이 패턴의 최적 환경. 학생마다 셔플로 다른 배치 판을 받고 각자 탭으로 풀며 자동 채점·자동 빙고 판정. 모바일 세로에서 판이 화면 폭에 꽉 차게.
- **TV 수업용**: 화면에 공동 판 하나를 띄우고 교사가 문제를 읽으면 학생이 정답 칸을 호명, 교사가 그 칸을 클릭해 마킹. 칸 라벨·문제 글자 크게(스크롤 없이 한 화면 수납, 4×4·5×5는 칸 글자 자동 축소).
- **전자칠판용**: 대형 판으로 학생이 직접 나와 정답 칸을 탭. 칸을 대형 터치 타겟으로, 마킹 도장 애니메이션을 크게.

## 검증 추가 포인트
- `cells` 길이 ≥ `bingoSize`² 인가 — 부족하면 판을 못 채움 (B6 연계)
- 모든 `questions[].answer`가 `cells`에 정확히 존재하는가 — 일치 칸이 없으면 절대 맞힐 수 없음 (A2·B6 연계)
- 줄 완성 판정이 가로·세로·**양 대각선**까지 정확한가 (대각선 누락이 단골 버그)
- `targetLines`가 도달 가능한가 — 출제 문항으로 마킹될 칸들이 적어도 한 줄을 만들 수 있는지 (A5 연계)
- 빙고 도달·문제 소진 두 종료 조건이 모두 결과 화면으로 정확히 귀결되는가, 중복 종료 없는가 (B5 연계)
- 이미 마킹된 칸 재탭·연타 시 줄 수 중복 집계 없음 (B4 연계)
