# 패턴 11 — 표적 사냥 (골라잡기)

화면에 흩어진 여러 항목 중 제시된 기준에 맞는 것만 빠르게 탭하는 실시간 변별 게임.

## 이런 자료에 적합
- **이분 판별**이 가능한 항목이 다수인 자료 — 맞춤법(맞다/틀리다), 영단어 철자(맞는 스펠링/틀린 스펠링)
- 사실/오개념 구분, 참/거짓 명제 가려내기
- 어떤 집합·범주에 **속함/안 속함** (예: 포유류만 잡기, 홀수만 잡기, 받침 있는 낱말만 잡기)
- 한 라운드에 6~12개 항목, 그중 맞는 항목이 2개 이상 섞이도록 뽑을 수 있을 때
- 04 스피드 챌린지(한 문제씩 즉답)와 달리, **여러 항목이 동시에 보이고 그중에서 골라야** 변별 효과가 사는 자료

## 게임 규칙
1. 라운드가 시작되면 상단에 기준(rule)이 뜨고, 그 기준에 맞는 항목과 안 맞는 항목이 섞여 화면에 배치된다.
2. 기준에 **맞는 항목(correct=true)**을 탭하면 +점, 잡힌 표시(사라지거나 체크). **안 맞는 항목**을 탭하면 감점 + 짧은 해설 오버레이로 "왜 기준에 안 맞는지(또는 무엇이 맞는지)"를 알려준다.
3. 그 라운드의 맞는 항목을 모두 잡거나 시간이 다하면 라운드 종료 → 다음 라운드. 모든 라운드 후 결과 화면: 점수·정확도, **놓친 정답 항목과 잘못 잡은 항목 다시 보기**.

## GAME_DATA 스키마
```javascript
const GAME_DATA = {
  title: "맞춤법 표적 사냥",       // 게임 제목
  grade: "초등 4학년 국어",        // 대상
  timeLimit: 25,                   // 라운드 제한 시간(초). 0이면 무제한(맞는 항목 다 잡으면 종료)
  motion: true,                    // true=항목이 떠다님 / false=격자 고정 배치
  rounds: [
    {
      rule: "맞춤법이 바른 낱말만 잡아라!",   // 이 라운드의 판별 기준
      items: [
        { text: "설거지", correct: true,  explain: "" },
        { text: "깨끗이", correct: true,  explain: "" },
        { text: "설겆이", correct: false, explain: "'설거지'가 바른 표기예요." },
        { text: "깨끗히", correct: false, explain: "부사 '깨끗이'는 '이'로 적어요." },
        { text: "며칠",   correct: true,  explain: "" },
        { text: "몇일",   correct: false, explain: "'며칠'이 바른 표기예요." }
      ]
    }
    // 라운드를 여러 개 둘 수 있어요. rule만 바꾸면 같은 메커니즘으로 다른 변별을 학습합니다.
  ]
};
```

## 구현 포인트
- 라운드마다 rule을 크게 표시하고, 그 라운드의 items를 **셔플해 배치**한다 (정답 항목이 늘 같은 자리에 오지 않게).
- 라운드가 여러 개일 때 **라운드 제시 순서는 매 판 셔플**한다. 단, **라운드를 난이도·주제 순으로 의도적으로 고정 배치한 경우는 고정 배치 예외**로 명시한다. 라운드 내부의 items 셔플은 이와 별개로 항상 적용한다.
- 정답 항목 탭: 잡힘 처리(페이드아웃 또는 체크 표시) + 점수 + 효과음. 오답 항목 탭: 감점 + 해당 항목의 explain을 짧은 오버레이로 띄워 변별 학습을 유도한다 (explain이 비면 rule을 다시 보여줌). **정답 항목의 explain은 비워도 되지만 오답 항목은 explain을 채우는 것을 원칙**으로 한다.
- 같은 항목 중복 탭 방지: 한 번 판정된 항목은 입력을 막는다(잡힘/오답 모두). 연타로 점수가 중복 가산되면 안 된다.
- **motion=true**: 항목 카드를 부드럽게 부유시킨다. 서로 겹쳐 글자가 가려지지 않게 간격을 유지하고, 화면 밖으로 나가면 경계에서 튕겨 들어오게 처리한다. 카드가 움직여도 탭 타겟이 손가락보다 작아지지 않게(전자칠판 최소 80×80px).
- **motion=false**: CSS Grid로 격자 고정 배치. 저학년·정밀 탭이 필요한 경우, 움직임이 어지러운 학생을 위한 안정 모드.
- **한 화면 수납**: 라운드의 모든 항목 카드 + rule + 타이머가 1280×800·1920×1080 두 해상도에서 스크롤 없이 한 화면에 들어오도록 항목 수에 따라 카드 크기·격자 열 수를 조정한다(@media max-height 압축, 임계값 1080 이상 — design-guide §5 D2 연계). 항목이 많아 한 화면을 넘으면 라운드를 분할한다.
- 라운드 종료 조건: 맞는 항목을 모두 잡았거나(timeLimit=0이면 이 조건만), 제한 시간이 0이 된 순간. 시간 종료 시 미처 못 잡은 정답 항목은 '놓친 항목'으로 기록한다.
- 결과: 총점·정확도(맞는 탭 / 전체 탭) + 라운드별로 **놓친 정답 항목**과 **잘못 잡은(오답) 항목**을 rule·정답기준과 함께 복습 목록으로 보여준다.
- 효과음은 design-guide의 표준 SoundEffect 유틸을 그대로 쓴다 (정답 탭=correct, 오답 탭=wrong).

## 활용 방식별 변형
- **전자칠판용**: 이 패턴의 최적 환경. 대형 터치로 학생이 직접 나와 몸으로 표적을 잡는다. motion=true로 두면 참여 열기가 높다. 카드를 크게, 라운드 rule은 화면 최상단에 항상 고정.
- **개별 접속용**: 각자 모바일에서 표적을 탭. motion은 기기 성능을 고려해 선택(저사양이면 false 권장). 정확도·놓친 항목 복습이 개인 학습에 강하다.
- **TV 수업용**: 학생들이 "저거!" 하고 외치면 교사가 클릭. timeLimit을 0으로 두고 학급이 함께 기준에 맞는 것을 골라가며 토론하는 방식이 어울린다 (속도 경쟁보다 변별 토의).

## 검증 추가 포인트
- correct 플래그로 정/오답 판정이 정확한가 — 정답 탭은 가산, 오답 탭은 감점으로 올바르게 분기 (A2 연계)
- 오답 항목 탭 시 "정답 기준이 무엇인지" 변별 해설이 뜨는가 (explain 또는 rule 재표시 — A5 연계)
- motion 모드: 항목 겹침으로 글자가 가려지지 않는가, 화면 밖 이탈 없이 경계 처리되는가, 연타·중복 탭 방지가 되는가 (B4 연계)
- 시간 종료 순간 입력이 차단되어 종료 후 탭이 점수에 반영되지 않는가 (B5 연계)
- 결과 화면에 놓친 정답 항목·잘못 잡은 항목 복습이 빠짐없이 모이는가 (D3 연계)
- GAME_DATA에 필수 공통 필드(title·grade) 외에 timeLimit·motion·rounds(rule/items/correct/explain)만 들어 있고 그 밖의 미사용 필드가 없는가 (B6 연계)
