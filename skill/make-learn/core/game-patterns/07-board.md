# 패턴 07 — 말판 보드게임

주사위를 굴려 말을 이동하고, 도착한 칸의 문제를 풀며 결승점을 향해 가는 게임.

## 이런 자료에 적합
- 복습에 **놀이성·운 요소**를 더하고 싶을 때 (실력 차가 큰 학급에서 모두가 이길 가능성)
- 문항 난이도가 고르게 섞인 자료, 또는 여러 소단원을 한 번에 복습할 때
- 모둠 활동(2~4팀)으로 운영할 때 — 전자칠판·TV 수업과 잘 맞음
- 문항 10~20개를 뽑을 수 있을 때

## 게임 규칙
1. 시작 화면에서 팀 수(2~4)·팀 말(이모지) 선택.
2. 차례 팀이 주사위를 굴려 말 이동 → 도착 칸 이벤트:
   - **문제 칸(대부분)**: 문제 표시 → 답하기(보기 선택 또는 교사 판정) → 맞으면 그 자리, 틀리면 2칸 후퇴 + 정답·해설 표시.
   - **이벤트 칸(소수)**: 앞으로 2칸, 뒤로 3칸, 한 번 쉬기, 다시 굴리기 등.
3. 먼저 결승점에 도착한 팀 승리 (또는 전원 도착까지 순위전). 결과 화면에서 틀린 문항 복습.

## GAME_DATA 스키마
```javascript
const GAME_DATA = {
  title: "게임 제목",
  grade: "초등 6학년 사회",
  boardSize: 24,             // 칸 수 (16~30 권장)
  judgeMode: "choice",       // "choice"(보기 선택 자동 채점) | "teacher"(교사 판정 — 구두 응답)
  events: { 3: "back2", 9: "again", 15: "rest", 20: "forward2" },  // 칸번호: 이벤트
  questions: [
    { q: "문제", choices: ["보기1","보기2","보기3","보기4"], answer: 0, explain: "해설" },
    // judgeMode "teacher"면: { q: "문제", a: "정답", explain: "해설" }
  ]
};
```

## 구현 포인트
- **말판 레이아웃**: 격자 둘레를 도는 사각 트랙(또는 S자 트랙)을 CSS Grid로 생성. 칸 번호와 이벤트 아이콘 표시. boardSize에 따라 자동 배치.
- 말 이동은 한 칸씩 순차 애니메이션 (이동 중 입력 잠금).
- 문제는 풀에서 셔플 순환 출제 (칸과 문제를 고정 매핑하지 않음 — 같은 칸 재도착 시 새 문제).
- 주사위는 굴림 애니메이션 후 결과 표시. 탭 한 번으로 굴리기.
- 진행 상태(말 위치·차례) localStorage 저장, 이어하기 지원 (수업 중단 대비).
- 후퇴 시 0번 칸 밑으로 내려가지 않게 경계 처리.

## 활용 방식별 변형
- **TV 수업용**: 교사가 조작, 모둠이 구두로 답함 → judgeMode "teacher" 권장.
- **전자칠판용**: 모둠 대표가 나와 직접 주사위 굴리고 보기 터치 → judgeMode "choice". 이 패턴의 최적 환경.
- **개별 접속용**: 1인 vs 컴퓨터(간단한 자동 상대) 또는 혼자 완주 기록 도전 모드로 변형. 우선순위 낮음 — 개별 접속용이면 다른 패턴을 먼저 추천.

## 검증 추가 포인트
- 결승점 정확 도달 처리 (초과 굴림: 정확히 멈추기 vs 도착 인정 — 명세서에 정한 규칙대로)
- 이벤트 칸 연쇄 (후퇴해서 또 이벤트 칸에 도착) 무한 루프 없음 (B5 연계)
- 이동 애니메이션 중 연타 잠금 (B4 연계)
- 말판 칸 수와 events 키 범위 일치 (B6 연계)
